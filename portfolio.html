Number guessing game

Project Overview

This game was created as a part of a school project with the main purpose to learn Javascript and DOM-manipulation. The project is hosted on Github-pages.

Game description

  • A random integer between 0-100 will be generated.
  • The user have 5 guesses to guess the correct number.
  • If the guessed number is too high or too low the user will notified with a popup.
  • If the user runs out of guesses without guessing the correct number a losing message will be displayed and the user has the option to restart the game and play again.
  • When the correct number is guessed a winning message will be displayed and the user has the option to restart the game and play again.

Car park monitor application

Project Overview

This project was the last part of the DevOps Engineering program. The main concept was to create some sort of application that uses a database to store its data. I chose to create an application with a graphical user interface for a car parking monitor. The app is built with Python, Tkinter and CustomTkinter. For the database I chose to work with SQLite. Total amount of parking spaces for this specific parking lot is 50 spots where each registration number can't be duplicated for one single parking session. The price is free of charge for the first 60 minutes, thereafter the price is set to 0.25 SEK/minute. The application has three main funtions: Start parking, See parked car status and Stop parking.

Main menu

The main menu gives the client three options: Start parking, See Parked Car Status and Stop parking. Apart from the functions there's a price list displayed along with time, date and availabe parking spots.

Start Parking

After clicking on start parking on the main menu the client gets to the page demonstrated on the first picture below. Here the client gets asked to enter a registration number consisting of 6 characters. If the registration number is already entered in the database the client will get an error and will be asked to enter another number. If the number is formated correct and is unique, the parking session will be successfully started, shown on the second picture.

Parked car Status

This function displays a summary for each unique registration number. The summary includes Start date and time, Parking status, Total parking time and Current price.

Stop parking

The Stop parking function stops the parking timer for the entered registration number and gives a summary over the parking session, shown on the first picture. When the parking is stopped the client continues to a ''payment'' prompt, seen on the second picture.

Rock paper scissors game

Project Overview

This game was created as a part of a school project with the main purpose to learn Javascript and DOM-manipulation. The project is hosted on Github-pages.

Game description

  • The playing area is divided in two halves, where the the player has the blue side while the CPU has the red side.
  • Each round starts when the player chooses ONE of three choices: Rock, Paper or Scissors.
  • Each won round gives 1 point to the winner of the round. A tied round gives no points.
  • The game ends when either the player or CPU reaches 5 points.
  • Rock > Scissors | Scissors > Paper | Paper > Rock.
